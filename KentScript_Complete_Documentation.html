üöÄ PART 1: INTRODUCTION

What is KentScript?

KentScript is a modern, powerful scripting language created by Musika Alvin at UICT Nakawa Institute. It uniquely combines the best features of three influential languages:

Python Simple, readable syntax perfect for beginners
Rust Memory safety with borrow checking
JavaScript Flexible, dynamic typing with async/await
Modern Production-ready with full ecosystem

Why KentScript Was Created

Traditional scripting languages face a fundamental problem: they are either slow or unsafe. KentScript solves this:

  • ‚ö° 5-10x Performance: Via bytecode compilation
  • ü¶Ä Memory Safety: Rust-style borrow checking
  • üì¶ Production Ready: Full ecosystem of 25+ modules
  • üîÑ Modern Async: Native async/await support
  • üßµ Multi-threading: True OS-level threading
üí° Pro Tip: KentScript is ideal for developers transitioning from Python to more performance-critical work. With 6+ years of Python experience, you'll find KentScript's syntax immediately familiar!

Ownership & Borrowing System

Understanding Ownership

Unique feature inspired by Rust that prevents memory bugs:

// Single owner
let x = 42;
let y = move x to y;  // Transfer ownership
print(y);             // 42 ‚úÖ
// print(x);          // ‚ùå Error: x no longer owns the value

Immutable Borrowing (Multiple Readers)

let data = [1, 2, 3, 4, 5];

// Multiple immutable borrows allowed!
let ref1 = borrow data;
let ref2 = borrow data;

print(ref1[0]);  // 1
print(ref2[2]);  // 3

release ref1;
release ref2;

Mutable Borrowing (Exclusive Access)

let mut counter = 0;

// Mutable borrow - exclusive!
let m = borrow *counter;

counter = counter + 1;  // Can modify
counter = counter + 1;

print(counter);  // 2

release m;
üéì Learning from Rust: If you're familiar with Rust's ownership model, you already understand KentScript's core safety feature! This brings compile-time safety to a Python-like language.

Performance & Bytecode Compilation

Why Bytecode?

KentScript can compile to bytecode for 5-10x performance improvement:

# Interpreter mode (flexible)
python kentscript.py script.ks

# Bytecode mode (fast!)
python kentscript.py script.ks --bytecode

Performance Example

func fibonacci(n) {
    if n <= 1 { return n; }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

let result = fibonacci(35);

Execution times:

  • Interpreter mode: ~2.3 seconds
  • Bytecode mode: ~0.23 seconds (10x faster!)
üí° When to use bytecode:
  • ‚úÖ Production deployments
  • ‚úÖ Data processing scripts
  • ‚úÖ Performance-critical code
  • ‚ùå Development/debugging (use interpreter)
  • ‚ùå Complex features like async/match

Optimization Tips

  • Use bytecode compilation for production
  • Prefer built-in functions over manual loops
  • Use list comprehensions instead of loops
  • Use ThreadPool for parallel processing
  • Cache computed values

class DataAnalyzer {

Cybersecurity Applications

Password Manager

import crypto;
import database;

class PasswordManager {
    func __init__(master_password) {
        self.db = database.connect("passwords.db");
        self.master_hash = crypto.sha256(master_password);
    }
    
    func add_password(service, username, password) {
        let encrypted = crypto.encrypt(password, self.master_hash);
        database.execute(self.db,
            "INSERT INTO passwords (service, username, password) VALUES (?, ?, ?)",
            [service, username, encrypted]
        );
    }
    
    func get_password(service) {
        let result = database.query(self.db,
            "SELECT password FROM passwords WHERE service = ?",
            [service]
        );
        if result.len() == 0 { return null; }
        return crypto.decrypt(result[0]["password"], self.master_hash);
    }
}

let pm = PasswordManager("MyMasterPassword!");
pm.add_password("gmail", "musika@gmail.com", "securepass123");

Game Development

Simple Game Engine

class GameObject {
    func __init__(x, y, width, height) {
        self.x = x;
        self.y = y;
        self.width = width;
        self.height = height;
        self.velocity_x = 0;
        self.velocity_y = 0;
    }
    
    func update() {
        self.x = self.x + self.velocity_x;
        self.y = self.y + self.velocity_y;
    }
    
    func render() {
        print("Object at (" + str(self.x) + ", " + str(self.y) + ")");
    }
    
    func collides_with(other) {
        return !(self.x > other.x + other.width or
                 self.x + self.width < other.x or
                 self.y > other.y + other.height or
                 self.y + self.height < other.y);
    }
}

class GameEngine {
    func __init__(width, height, fps) {
        self.width = width;
        self.height = height;
        self.objects = [];
    }
    
    func add_object(obj) {
        self.objects.append(obj);
    }
    
    func update() {
        for obj in self.objects {
            obj.update();
        }
    }
    
    func render() {
        for obj in self.objects {
            obj.render();
        }
    }
}

let game = GameEngine(800, 600, 60);
let player = GameObject(400, 550, 20, 20);
game.add_object(player);

Debugging & Troubleshooting

Common Issues

Stack Underflow in Bytecode

Problem: Bytecode encounters unsupported expression.

Solution: Use interpreter mode or upgrade:

# Use interpreter instead
python kentscript.py script.ks

Memory Issues

Problem: Large data structures cause memory errors.

Solution: Use generators for large datasets:

// ‚ùå Uses lots of memory
let all_numbers = [x for x in range(0, 1000000)];

// ‚úÖ Memory efficient
func number_generator() {
    for i in range(0, 1000000) {
        yield i;
    }
}

Q: How is KentScript different from Python?

A: KentScript adds Rust-like borrow checking, bytecode compilation, and better performance while keeping Python's simplicity.
